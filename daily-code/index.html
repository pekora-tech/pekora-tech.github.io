開發日記

透過MCP自動化工具，將日常開發活動轉換為結構化的時間分割內容

功能定位

核心價值

  • 開發軌跡記錄：以時間為主軸記錄每日開發活動
  • 經驗快速捕捉：即時記錄開發過程中的發現與問題解決
  • 知識自動整合：透過MCP工具自動整理並生成日報

與其他模組的關係

daily-code → projects：日常經驗整合為深度分析
daily-code → component：頻繁模式提煉為可重用元件
daily-code → doc：操作經驗沉澱為標準化文件

MCP自動化工作流程

設計架構

指定資料夾掃描 → commit項目識別 → 標題描述生成 → 自動commit → 日報整合

實作策略

  1. claude/gemini cli驅動：透過AI工具自動化處理
  2. 資料夾監控：掃描指定目錄下的待commit項目
  3. 智能生成：為每個變更自動產生合適的title與description
  4. 批次處理：完成所有commit後整合為單日文件

安全機制

原始記錄層：完整技術細節（內部保留）
處理過濾層：敏感內容標記（自動化檢查）
發布展示層：安全內容輸出（對外可見）

內容組織策略

命名規範

  • 檔案格式：YYYY-MM-DD-主題關鍵字.md
  • 標題結構：日期 + 核心技術 + 主要成果
  • 標籤系統：技術棧 + 專案類別 + 學習類型
