GitHub 開發者精要指南 🚀

專精收斂而非擴散 - 聚焦最實用的 GitHub 工作流程

🏷️ Commit Message 規範（核心！）

標準格式

<type>: <description>

範例：
feat: 新增用戶登入功能
fix: 修正購物車計算錯誤
docs: 更新 API 使用說明
style: 統一程式碼格式
refactor: 重構資料庫查詢邏輯

快速參考

Type用途範例
feat✨ 新功能feat: 新增深色模式切換
fix🐛 修復錯誤fix: 修正登入驗證失效問題
docs📚 文檔更新docs: 更新安裝說明
style🎨 格式調整style: 修正程式碼縮排
refactor♻️ 程式碼重構refactor: 優化資料處理邏輯
perf⚡ 效能優化perf: 改善查詢效能
chore🔧 維護更新chore: 更新依賴套件

🌊 Git 工作流程

GitHub Flow（推薦）

graph LR
    A[main] --> B[創建 feature branch]
    B --> C[開發功能]
    C --> D[Push & PR]
    D --> E[Code Review]
    E --> F[Merge to main]
    F --> G[自動部署]

📝 Pull Request 最佳實踐

PR 標題格式

[Type] 簡潔描述

範例：
[Feature] 新增用戶個人資料編輯
[Fix] 修正購物車總價計算錯誤
[Docs] 更新部署說明文檔

PR 描述模板

📋 Issue 管理

Issue 標題格式

[類型] 具體描述問題

範例：
[Bug] 登入後頁面白屏無法載入
[Feature] 希望新增深色模式支援
[Question] API 速率限制相關問題

常用標籤系統

標籤顏色用途
bug🔴錯誤回報
enhancement🟢功能改進
documentation🔵文檔相關
good first issue🟡新手友善
priority: high🔴高優先級

📁 Repository 結構標準

必備檔案清單

project-root/
├── 📄 README.md              # 專案說明
├── 📄 LICENSE                # 授權條款
├── 📄 .gitignore            # 忽略規則
├── 📄 CHANGELOG.md          # 版本記錄
├── 📁 .github/
│   ├── workflows/            # CI/CD 配置
│   ├── ISSUE_TEMPLATE/       # Issue 模板
│   └── PULL_REQUEST_TEMPLATE.md
├── 📁 src/                   # 原始程式碼
├── 📁 docs/                  # 技術文檔
└── 📁 tests/                 # 測試檔案

🏷️ 版本管理（Semantic Versioning）

版本號規則

MAJOR.MINOR.PATCH

1.0.0 → 1.0.1  (patch: 修復錯誤)
1.0.1 → 1.1.0  (minor: 新增功能)
1.1.0 → 2.0.0  (major: 重大變更)

👥 Code Review 指南

Review 重點項目

  • ✅ 功能正確性：是否達成預期目標
  • ✅ 程式碼品質：可讀性和維護性
  • ✅ 效能考量：是否有效能瓶頸
  • ✅ 安全性：是否有潛在風險
  • ✅ 測試覆蓋：是否有充分測試

良好的 Review 評論

✅ 建設性評論：
"建議在第 15 行加入錯誤處理，避免 null 的情況"
"這個邏輯很清晰！可以考慮提取成獨立函數增加復用性"
"程式碼風格很棒，邏輯也沒問題"

❌ 無效評論：
"不對"
"改掉"
"這樣不好"

📊 Repository 健康度指標

品質徽章

定期檢查項目

  • 依賴套件安全性更新
  • CI/CD 流程運行狀況
  • Issue 回應時間是否合理
  • 文檔是否保持最新
  • 程式碼測試覆蓋率

🎯 行動清單

新專案設定（必做）

  • 設定 commit message 規範
  • 建立 .gitignore 檔案
  • 撰寫清晰的 README.md
  • 設定基本的 GitHub Actions
  • 建立 Issue 和 PR 模板

進階優化（選做）

  • 整合程式碼品質檢查
  • 設定自動化測試流程
  • 建立完整的 CI/CD pipeline
  • 設定依賴套件安全掃描

💡 核心原則

  • 🎯 保持一致性：統一的命名和格式規範
  • ⚡ 自動化優先：能自動化的絕不手動操作
  • 📚 文檔同步：程式碼變更時同步更新文檔
  • 🔄 持續改進：根據團隊反饋優化流程

記住：工具是為了提高效率，不要被流程束縛創造力！
