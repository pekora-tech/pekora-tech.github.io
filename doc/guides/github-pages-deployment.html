從手動到自動：建立專業級的 GitHub Pages 部署流程

🎯 核心架構

雙倉庫分離設計

graph TB
    A[📝 selfpagesource<br/>原始碼 + 內容] -->|GitHub Actions| B[🌐 username.github.io<br/>純靜態網站]

    A1[Markdown 內容] --> A
    A2[Quartz 配置] --> A
    A3[CI/CD 腳本] --> A

    B --> C[🚀 線上網站<br/>https://username.github.io]

    B1[HTML/CSS/JS] --> B
    B2[靜態資源] --> B

設計原則：

  • 🔒 關注點分離：原始碼與發布內容隔離
  • ⚡ 自動化優先：推送即部署，零手動操作
  • 🧹 保持純淨：發布倉庫只含靜態檔案

🎯 最佳實踐

安全性

  • 🔒 Token 管理：定期更新，使用最小權限
  • 🔒 敏感資訊：永不提交 .env 檔案
  • 🔒 權限控制：原始碼倉庫可設為私有

維護性

  • 📅 定期檢查：Token 過期提醒
  • 📊 監控指標：部署成功率、建置時間
  • 📚 文檔更新：配置變更時同步更新 SOP

效率性

  • ⚡ 本地測試：使用 npm run preview 預先驗證
  • ⚡ 增量部署：只在內容變更時觸發
  • ⚡ 快速回滾：保留 Git 歷史便於恢復

📝 檢查清單

初次設定

  • 建立 {username}.github.io 倉庫
  • 生成 Personal Access Token
  • 設定 DEPLOY_TOKEN Secret
  • 配置 .github/workflows/deploy.yml
  • 測試首次部署

日常維護

  • Token 是否即將過期？
  • Actions 執行是否正常？
  • 網站載入速度如何？
  • 建置時間是否合理？
  • 錯誤日誌是否有異常？

核心理念：工具是為了提高效率，一次設定，永久受益！
