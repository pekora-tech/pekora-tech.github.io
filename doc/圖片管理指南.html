🖼️ 複製貼上圖片的標準流程

方法一：VSCode + Paste Image 擴展 (推薦)

  1. 安裝VSCode擴展

    • 安裝 “Paste Image” 擴展
    • 設定圖片存放路徑

  2. VSCode設定檔配置 (settings.json)

  3. 使用流程

    • 複製圖片到剪貼板 (Cmd+C)
    • 在Markdown檔案中按 Cmd+Shift+V
    • 自動產生相對路徑連結

方法二：手動管理 (精確控制)

  1. 圖片命名規則

    {專案名稱}-{功能描述}-{序號}.{副檔名}

範例：
test-ai-memory-architecture-01.png
test-ai-memory-demo-screenshot-02.png
test-ai-memory-api-swagger-03.png

  2. 存放位置

    專案：test-ai-memory.md
圖片：assets/images/ai-tools/test-ai-memory/

  3. Markdown引用語法

🌐 網頁生成時的路徑處理

相對路徑最佳實踐

從專案文檔到圖片的路徑計算：

專案文檔位置：content/projects/ai-tools/test-ai-memory.md
圖片位置：    content/assets/images/ai-tools/test-ai-memory/demo.png
相對路徑：    ../../assets/images/ai-tools/test-ai-memory/demo.png

路徑計算公式：

  • 從 projects/{domain}/{project}.md
  • 到 assets/images/{domain}/{project}/image.png
  • 需要 ../../assets/images/{domain}/{project}/image.png

靜態網站生成器相容性

Hugo/Jekyll/Gatsby等靜態生成器

  • ✅ 相對路徑在大多數情況下都能正常工作
  • ✅ assets/ 資料夾通常會被複製到網站根目錄
  • ✅ 圖片URL會變成 /assets/images/...

注意事項

  • 確保圖片檔案小於5MB (網頁載入速度)
  • 使用WebP格式獲得更好的壓縮率
  • PNG用於截圖，JPG用於照片

🎯 最佳實踐原則

  1. 命名規範

    • 使用英文和數字
    • 用連字符 - 分隔單詞
    • 包含專案名稱前綴

  2. 檔案大小控制

    • 截圖：<500KB
    • 示意圖：<200KB
    • 考慮使用WebP格式

  3. 組織原則

    • 按專案分類存放
    • 與文檔結構保持一致
    • 定期清理無用圖片

  4. 網頁相容性

    • 使用相對路徑
    • 確保圖片alt text描述
    • 考慮響應式設計

⚠️ 注意事項

  • 不要使用絕對路徑 /Volumes/Code/...
  • 不要使用中文檔名
  • 不要放置過大的圖片檔案
  • 確保圖片版權清晰
  • 定期檢查圖片連結有效性

遵循此指南，確保圖片管理的系統性、一致性和可維護性。
