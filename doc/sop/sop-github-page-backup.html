GitHub Pages + CI/CD 自動部署 SOP

“從零到自動化：建立專業級的 GitHub Pages 部署流程”

🎯 目標與架構設計

第一性原理思考

  • 核心需求：程式碼管理 + 自動建置 + 公開發布
  • 分離原則：原始碼倉庫 ≠ 發布倉庫，保持關注點分離
  • 自動化優先：一次推送，自動部署，解放重複勞動

雙倉庫架構

📦 selfpagesource (私有/公開原始碼)
├── 📝 內容撰寫與管理
├── 🔧 Quartz 配置與建置
└── 🚀 CI/CD 自動化腳本

⬇️ GitHub Actions 自動部署

📦 username.github.io (公開發布)
├── 📄 純靜態 HTML 檔案
├── 🎨 CSS/JS 資源
└── 🌐 直接對外服務

📋 完整建置流程

Phase 1: 建立發布倉庫 (username.github.io)

1.1 創建 GitHub Pages 倉庫

  1. 登入 GitHub → 點擊右上角 + → New repository
  2. 倉庫命名規則：必須是 {username}.github.io 
    範例：pekora-tech.github.io
⚠️ 必須與您的 GitHub 用戶名完全一致
  3. 基本設定：
    • ✅ Public（必須公開才能使用 GitHub Pages）
    • ✅ Add a README file
    • ❌ 不需要 .gitignore（會自動部署覆蓋）
    • ❌ 不需要 License（由原始碼倉庫管理）

1.2 啟用 GitHub Pages

  1. 進入倉庫 → Settings → 左側選單 Pages
  2. Source 設定：
    • Source: Deploy from a branch
    • Branch: main
    • Folder: / (root)
  3. 自定義域名（可選）：
    • 若有自己的域名，在 Custom domain 填入
    • 建議先用預設的 username.github.io 測試

1.3 驗證部署

  • 等待 1-2 分鐘後訪問：https://username.github.io
  • 應該看到預設的 README 內容

2.1 初始化專案倉庫

2.2 連接到 GitHub

    • 倉庫名稱：selfpagesource（或您喜歡的名稱）
    • 可設為 Private（原始碼不一定要公開）
    • 不初始化檔案（本機已有）

  2. 本機連接遠端：

2.3 設定 .gitignore

Phase 3: 設定 Personal Access Token

3.1 生成 GitHub Token

  1. GitHub 右上角頭像 → Settings
  2. 左側選單最下方 → Developer settings
  3. Personal access tokens → Tokens (classic)
  4. Generate new token → Generate new token (classic)
  5. Token 設定：
    • Note: Deploy to GitHub Pages
    • Expiration: No expiration（或設定適當期限）
    • Scopes: ✅ repo（完整倉庫權限）

⚠️ 重要：複製 Token 並妥善保存，頁面關閉後無法再次查看

3.2 設定 Repository Secret

  1. 進入 原始碼倉庫（selfpagesource）
  2. Settings → Secrets and variables → Actions
  3. New repository secret：
    • Name: DEPLOY_TOKEN
    • Secret: 貼上剛才複製的 Personal Access Token
  4. Add secret 完成

5.1 提交初始程式碼

  3. 觀察每個步驟的執行狀況：
    • ✅ Checkout 原始碼

  • 訪問 https://username.github.io
  • 應該看到您的 Quartz 網站內容

常見問題與解決方案

❌ Problem 1: Actions 執行失敗 - Permission denied

症狀：

sh: 1: quartz: Permission denied
Error: Process completed with exit code 127

解決方案： 已在 workflow 中加入權限修正步驟：

❌ Problem 2: Personal Access Token 無效

症狀：

remote: Permission to username/username.github.io.git denied
fatal: unable to access 'https://github.com/': The requested URL returned error: 403

解決方案：

  1. 檢查 Token 是否過期
  2. 確認 Token 有 repo 權限
  3. 重新生成 Token 並更新 Secret

❌ Problem 3: 部署後網站顯示 404

症狀：訪問 username.github.io 顯示 404 錯誤

解決方案：

  1. 確認 GitHub Pages 設定正確（Source: main branch）
  2. 檢查 public/ 目錄是否有 index.html
  3. 等待 5-10 分鐘讓 GitHub Pages 更新

❌ Problem 4: 格式檢查失敗

症狀：

[warn] Code style issues found in the above file(s). Forgot to run Prettier?

解決方案： 提交前執行：

❌ Problem 5: 建置記憶體不足

症狀：

📊 維護與監控

定期檢查清單

  • 每月檢查 Personal Access Token 是否即將過期
  • 確認 Actions 執行歷史無異常
  • 監控網站載入速度與可用性
  • 更新 Node.js 版本與依賴套件
  • 備份重要內容與設定

安全性考量

  • 🔒 定期更新 Token 權限
  • 🔒 不在程式碼中暴露敏感資訊
  • 🔒 監控 Actions 執行日誌
  • 🔒 使用最小權限原則
