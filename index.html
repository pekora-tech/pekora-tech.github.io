SelfPageSource 技術文件化專案

透過系統分析師視角，將專案理解轉換為可持續的開發歷程文件

專案核心理念

第一性原理思考

  • 資訊整理本質：從零散經驗提煉出可重複利用的知識體系
  • 知識傳承需求：將個人經驗轉化為團隊可共享的標準化文件
  • 安全平衡策略：在開放分享與創新保護之間找到最佳平衡點

KISS原則與高內聚低耦合

  • 採用Quartz 4.0作為靜態網站生成器，避免重新造輪子
  • 內容組織遵循功能分離，各模組職責明確
  • 自動化流程減少人工干預，專注於內容創作

內容架構導覽

技術文件中心

標準化技術文件與操作程序

功能定位：補齊技術資料的標準化文件庫 適用場景：技術操作標準程序、系統配置、開發工具指南、架構設計文件 文件特性：結構化內容、版本控制、標準格式

專案建置項目

專案主軸建置文件，支援多專案共存管理

功能定位：專案理解與分析的標準化記錄 管理原則：專案間關係明確標示、建置流程標準化、專案狀態可追蹤 安全機制：技術方法可公開，實作細節選擇性公開，創新核心嚴格保密

時間分割內容

以時間軸組織的日常開發記錄

功能定位：日常開發經驗的系統化記錄 重要原則：commit細寫記錄開發細節，對外文件需過濾敏感資訊 MCP整合：透過claude或gemini cli自動整合每日內容

可重用元件庫

從歷史創作中萃取有價值的可重用元件

功能定位：長遠計劃，建立技術資產庫 價值體現：減少重複開發、提升開發效率、形成個人技術標準 智財保護：展示思考過程，保護實作細節 最新元件：AI長期記憶系統 - 可應用於Helpdesk、客服系統

系統分析師工作流程

專案理解階段

  1. 讀取README.md：理解專案原始設計與目標
  2. 架構分析：識別技術棧與依賴關係
  3. 流程梳理：分析開發與部署流程
  4. 問題識別：發現潛在風險與改進點

文件轉換階段

  1. 內容分類：依據功能特性分配至適當目錄
  2. 標準化處理：統一文件格式、命名規則、標籤系統
  3. 安全審查：識別敏感內容、應用保密策略、確保合規發布

持續維護階段

  1. 定期回顧：文件有效性檢查、技術棧更新同步、流程優化改進
  2. 價值萃取：識別可重用元件、建立最佳實踐、形成技術標準

安全與保密機制

分級公開策略

  • 技術方法層級：可公開分享，促進技術交流
  • 實作細節層級：選擇性公開，保護競爭優勢
  • 創新核心層級：嚴格保密，維護智財價值

自動化檢查流程

  • 關鍵字過濾機制
  • 敏感內容標記系統
  • 發布前審查程序

技術架構

核心技術棧

  • Quartz 4.0：靜態網站生成器
  • Node.js：執行環境
  • TypeScript：類型安全開發
  • GitHub Pages：免費網站託管

自動化流程

專案分析 → 內容分類 → 標準化處理 → 安全審查 → 自動發布
