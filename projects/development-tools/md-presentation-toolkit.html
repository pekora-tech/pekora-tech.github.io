📦 MD Presentation Toolkit - 統一簡報生成平台

架構革新：從單一工具進化為 Monorepo 統一平台

支援 Slidev 和 MDX-Deck 雙引擎，實現真正的技術靈活性

專案概述

  • 路徑：/Volumes/Code/test-ppt-page/md-presentation-toolkit/
  • 技術棧：Node.js + OpenAI + MCP + Slidev + MDX-Deck + Express + Vue.js + Monorepo
  • 狀態：✅ Monorepo 轉換完成
  • 複雜度：🔥 高度複雜（多引擎整合 + Monorepo 架構）
  • 領域：開發工具與效率提升

背景與問題

從單一工具到統一平台的演進

原有 test-slidev 的局限：

  • 🔒 單一引擎綁定：只支援 Slidev
  • 🔧 重複開發：MDX-Deck 需要獨立實現
  • 📂 資源分散：共用功能無法重複利用
  • 🚧 維護困難：多專案版本不同步

Monorepo 統一平台的優勢

🏗️ 架構統一化：

  • 📦 統一依賴管理：消除重複依賴，減少衝突
  • 🔄 代碼重用：共用後端服務和工具函數
  • ⚡ 簡化開發流程：統一指令介面和開發環境
  • 🎯 獨立部署：各應用可獨立建構和部署

🎭 雙引擎支援：

  • 🎨 Slidev：Vue.js 生態，豐富互動功能
  • 📄 MDX-Deck：React 生態，JSX 語法支援
  • 🔀 無縫切換：相同資料來源，不同呈現方式
  • 🎯 用戶選擇：根據需求選擇最適合的引擎

技術方案

🌟 Monorepo 架構設計

1. 工作區管理策略

管理優勢：

  • ✅ 統一版本管理，避免依賴衝突
  • ✅ 共用 node_modules，節省磁碟空間
  • ✅ 跨 package 腳本執行
  • ✅ 並行建構和測試

2. 雙引擎整合架構

flowchart TD
    User[👤 用戶需求] --> Manager[🎯 引擎選擇器]
    
    Manager --> |選擇 Slidev| SlidevApp[🎨 Slidev 應用<br/>Vue.js 生態]
    Manager --> |選擇 MDX| MDXApp[📄 MDX-Deck 應用<br/>React 生態]
    
    subgraph shared [🔧 共用服務層]
        API[⚙️ Express API]
        Frontend[🎨 管理介面]
        MCP[🧠 MCP 整合]
    end
    
    subgraph assets [📂 共用資源]
        Data[📊 數據檔案]
        Images[🖼️ 圖片資源]
        Configs[⚙️ 配置檔案]
    end
    
    SlidevApp --> shared
    MDXApp --> shared
    shared --> assets
    
    SlidevApp --> Output1[🎪 Slidev 簡報<br/>:3030]
    MDXApp --> Output2[📱 MDX 簡報<br/>:8080]

架構特色：

  • 🔧 引擎抽象：統一介面，不同實現
  • 📊 資源共享：相同數據，多種呈現
  • 🎯 獨立運行：各引擎互不干擾
  • 🔄 無縫切換：保持相同的 MCP 和 AI 功能

3. 共用服務設計

共用功能：

  • 🧠 MCP 整合服務：保持原有 AI 功能
  • 📊 數據處理引擎：統一數據格式轉換
  • 🎨 前端管理界面：兩個引擎共用
  • ⚙️ 配置管理系統：統一設定檔管理

🚀 開發流程優化

統一指令系統

開發優勢：

  • ⚡ 快速啟動：一個指令啟動整個開發環境
  • 🔄 並行開發：同時開發兩個引擎應用
  • 🧪 統一測試：跨 package 測試執行
  • 📦 一鍵部署：統一建構和部署流程

💡 技術洞察與突破

1. Monorepo 最佳實踐

工作區管理洞察：

  • ✅ 依賴提升：共用依賴在根目錄，專用依賴在子 package
  • ✅ 腳本統一：根層級腳本控制所有 workspace 操作
  • ✅ 版本同步：使用 workspace protocol 確保版本一致
  • ✅ 建構優化：利用 npm workspace 並行執行

2. 雙引擎整合策略

抽象層設計：

關鍵成功因素：

  • 🎯 統一介面：不同引擎使用相同的 API
  • 📊 數據標準化：制定統一的數據格式規範
  • 🔧 配置抽象：引擎特定配置與通用配置分離
  • 🚀 漸進遷移：保持向後相容，逐步優化

3. 開發體驗提升

統一開發環境：

  • 🛠️ 工具鏈統一：ESLint、Prettier、TypeScript 配置共享
  • 🧪 測試策略：Jest 配置複用，測試工具統一
  • 📚 文檔生成：自動化 API 文檔和使用指南
  • 🔍 偵錯體驗：統一的日誌格式和錯誤處理
